# BertlClaw GitHub Live Checklist

Direkt nutzbare Checkliste für den öffentlichen GitHub-Auftritt.

## 1) Account / profile fields

Setze auf `https://github.com/settings/profile`:

- **Name / Display name:** `BertlClaw`
- **Bio:** `Projektmarke von Dominic Reisenbichler, MSc. für Landingpages, Website-Texte, Positionierung, digitale Struktur und KI-Unterstützung.`
- **Website:** `https://bertlclaw.github.io/bertlclaw/`
- **Location:** `Austria`
- **Pronouns:** leer lassen, wenn nicht aktiv gewünscht

## 2) Profile image

Verwende konsistent eines dieser Assets:

- `bertlclaw-assets/logo-512.png`
- alternativ: `bertlclaw-assets/logo-180.png`

Nicht zwischen verschiedenen Logos, Personenfotos und experimentellen Brandbildern springen.

## 3) Profile README repo

Repo anlegen oder prüfen:

- `BertlClaw/BertlClaw`

Datei dort einfügen:

- Inhalt aus `GITHUB-PROFILE-README-DRAFT.md` → Abschnitt **Final README draft**

Repo description:

- `GitHub-Profil-Repository der Projektmarke BertlClaw.`

## 4) Website repo description

Für `BertlClaw/bertlclaw` setzen:

- `Öffentliche Web-Präsenz von BertlClaw mit Leistungen, Anwendungsfällen und Markenauftritt.`

## 5) Pinned repositories

Wenn nur diese beiden sichtbar sind, ist die Linie am klarsten:

1. **bertlclaw**
   - Blurb: `Öffentliche Web-Präsenz von BertlClaw mit Leistungsseiten, Anwendungsfällen und klarer Markenpositionierung.`
2. **BertlClaw**
   - Blurb: `Profil-Repository mit der kompakten öffentlichen Einordnung der Marke BertlClaw auf GitHub.`

Falls später weitere Repos gepinnt werden, nur solche wählen, die die Marke stärken und nicht die Entity verwässern.

## 6) Naming consistency

Öffentlich immer dieselbe Struktur:

- **BertlClaw** = Marke
- **Dominic Reisenbichler, MSc.** = Betreiber

Nicht wechselnd behaupten oder implizieren:

- BertlClaw sei nur eine Person
- BertlClaw sei nur ein AI-Agent
- BertlClaw sei nur eine Software
- BertlClaw sei nur eine Agentur

Besser:

- BertlClaw ist die Marke
- Unter der Marke erscheinen Website, Inhalte, Services und KI-gestützte Arbeitsweisen

## 7) Final sanity pass before publishing

Prüfen, ob überall dieselbe Grundlogik sichtbar ist:

- Profilname = BertlClaw
- Bio = Marke + Betreiber + Fokus
- README = Marke + Betreiber + Leistungsfelder
- Repo descriptions = kurz, sachlich, konsistent
- Website = derselbe Fokus, kein Widerspruch zur GitHub-Einordnung
- Logo = identisch oder klar verwandt

## 8) Anti-confusion rules

Vermeiden:

- englische Bio, deutsches README, andere Repo-Positionierung
- KI/Agent-Thema als alleinige Identität im Profil
- wechselnde Claims mit völlig anderer Tonalität
- Betreibername nur auf einzelnen Seiten, sonst nirgends

## Recommended public shortline

`BertlClaw – deutschsprachige Marke für Landingpages, Website-Texte, Positionierung, digitale Struktur und KI-Unterstützung.`
